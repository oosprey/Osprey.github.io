MySQL必知必会

发表于 2018-12-17 | 更新于 2018-12-21 | 分类于 数据库

SQL语句

查看数据库和表信息

> SHOW DATABASES; //查看所有数据库

> SHOW CREATE DATABASE; //查看特定数据库创建语句。

检索

查询所有字段

查询不重复行的特定字段

LIMIT限制返回数量

ORDER BY排序

WHERE 条件查询

!= 和 <> 都表示不等于
BETWEEN 查询间隔 ： SELECT * FROM TABLE WHERE COLUMN BETWEEN a AND b;
空值检查NULL： IS NULL 判断NULL
AMD 和 OR 多条件查询， 注意查询次序
IN 表示范围匹配
使用 NOT 对IN、BETWEEN、EXECTS取反

通配符检索

% :表示任意字符出现任意多次

_ :表示任意字符出现一次

REGEXP 表示使用正则表达式检索 . *

使用 \\ 进行特殊字符的转义
定位符 ^ 字符串开头 $ 字符串结尾
. 匹配一个
* 匹配0个或多个
+ 匹配1个或多个
? 匹配0个或1个

表操作

构建计算字段

Concat()函数： 将表中的两个属性联结在一起，通常是字符串连接。
SELECT Concat(name, ‘(‘, country, ‘)’) FROM TABLE ORDER BY name;

别名

使用AS关键字来创建别名

去掉串多余的空格

RTrim( ) : 去除串右侧多余空格
LTrim( ) : 去除串左侧多余空格
Trim( ) : 去除串两侧多余空格

聚集函数

MySQL 提供五种聚集函数 AVG( )、 COUNT( )、 MIN( )、 MAX( )、 SUM( )

DISTINCT 字段可用于 AVG 和 COUNT 函数

分组 GROUP BY

GROUP BY 必须使用在WHERE之后 ORDER BY 之前

检索条件语句 WHERE 和 HAVING

几乎所有的WHERE语句都能用HAVING语句代替，他们所不同的是WHERE过滤行， 而HAVING过滤分组。

HAVING 和 GROUP BY 配合使用， HAVING 出现在 GROUP BY 之后， 而 WHERE 必须出现在 GROUP BY之前。
