操作系统

发表于 2018-12-20 | 分类于 操作系统

虚拟存储器、虚拟内存、交换空间、物理内存

虚拟存储器是对物理存储器的抽象，允许程序申请大于实际物理存储的内存，提供一致性的地址空间。

  • CPU生成虚拟地址，MMU进行地址翻译，发现缺页，物理内存不足，进行页置换。

    局部性原理

时间局部性

如果一条指令被执行或者某个数据被访问过，那么不久以后该指令可能再次执行该数据可能再次被访问

空间局部性

如果某个存储单元被访问过，那么不久之后他周围的存储单元也会被访问

虚拟内存技术的支持

  1. 一定量的内存和外存
  2. 页表机制
  3. 中断机构
  4. 地址变换机构

页面置换算法

  1. FIFO先进先出：优先置换最早进入内存的。
  2. LRU最近最久未使用算法：置换最近最长时间未访问过的页面。

进程管理

进程是为了实现操作系统的并发性和共享性

相关概念

  • 操作系统通过进程控制块PCB来控制和管理进程，PCB描述了进程的状态和基本情况
  • PCB、程序段、相关数据段一起构成了进程映像（实体），进程映像是静态的，进程是动态的。
  • 创建和撤销进程实际上就是创建和撤销PCB
  • 进程是程序的一次执行过程，所以进程是动态的，同时它是系统进行资源分配和调度的一个独立单位。

进程的状态

  • 运行
  • 就绪
  • 阻塞
  • 创建
  • 结束
  • 挂起

进程的控制（原语）

进程创建

  • 分配PID，申请新的PCB（PCB是有限的），申请失败则创建失败。
  • 分配相关资源，资源有限，没有空闲资源分配则进程进入阻塞状态等待资源分配
  • 初始化PCB，包括优先级、各种状态信息等。
  • 插入到就绪队列，等待调度

    进程终止

    正常终止和异常终止
  • 检查PCB状态，正在执行则立即停止
  • 终止所有子进程
  • 回收所有资源，或返还操作系统或返还父进程
  • 删除PCB

    进程切换

  • 保存处理机上下文，包括程序计数器和其他寄存器
  • 更新PCB信息
  • 将PCB移入相应的队列
  • 选择另一个PCB，更新其状态
  • 恢复上下文

进程间通信

  1. 共享存储：共享内存
  2. 消息传递：通过操作系统的消息传递
  3. 管道通信：共享缓冲区，半双工，同一时刻只能读或者写。

线程管理

线程：用户级线程：应用程序管理 内核级线程：内核控制

处理机调度

调度层次

  1. 作业调度：从外存中调入即将。
  2. 中级调度：进程挂起，调出内存，存入外存。
  3. 进程调度：低级调度，从就绪队列调出进程。

进程调度方式

  1. 非抢占式
  2. 抢占式

调度算法

  1. FCFS 先来先服务算法（非抢占式）
  2. SJF 短作业优先（对长作业不利）
  3. 优先级调度（抢占和非抢占）
  4. 高响应比优先调度算法（主要用于作业调度）
  5. 时间片轮转调度算法
  6. 多级反馈队列算法（时间片和优先级算法的综合发展）

进程同步

临界资源

临界资源是指一次只允许一个进程访问的资源，访问临界资源的代码被称作临界区。

信号量解决同步和互斥

整形信号量和记录型信号量

经典进程同步问题

  1. 生产者消费者问题
  2. 读者写者问题：有一个写其他就不能读或者写
  3. 哲学家进餐问题：两种解决方案: 1. 信号量：只允许同时四个哲学家进餐。 2. 只允许同时拿起左右两把叉子，否则一个都不拿。
  4. 吸烟者问题

死锁：多个进程因争抢资源而互相等待无法解开

死锁的条件

  1. 互斥条件： 进程占有的资源不可剥夺
  2. 请求和保持： 进程占有一个资源，申请另一个资源时被阻塞，而又不释放自己的资源。
  3. 循环等待： 存在进程等待链。

死锁处理策略

  1. 预防死锁

    破坏死锁产生的条件

  2. 避免死锁

    安全状态：系统不会发生死锁。预防死锁的方法要求系统在分配资源之前计算分配此资源是否会导致系统进入不安全状态。

银行家算法：每次进程申请资源时，系统判断当前已分配给进程的资源加上要申请的资源是否超过该进程最大需求量，超过则不分配，否则判断当前可用资源是否大于申请资源数，大于则分配，否则不予分配。
