进程复习

进程状态

  1. 创建
  2. 就绪
  3. 运行
  4. 阻塞
  5. 终止

进程终止过程

  1. 标志该进程未结束，等待资源回收
  2. 读取进程PCB，检查进程状态
  3. 若为运行状态，立即停止运行，并标志结束为真
  4. 检查是否有子孙进程，停止所有子孙进程
  5. 将资源全部释放给系统或父进程
  6. 移除队列

特殊进程

僵尸进程

是指子进程结束，父进程没有回收子进程的PCB，导致子进程仍存在系统中，该子进程称为僵尸进程

孤儿进程

父进程退出，子进程仍在运行，子进程失去管理者，称为孤儿进程。

守护进程

由用户终端创建，脱离该控制终端后运行于系统后台，运行周期较长，控制权交由系统管理的用户进程。

守护进程创建

  1. 创建子进程，父进程手动退出。
  2. 子进程中调用setid创建会话，目的是防止1号进程将该进程当做孤儿进程回收掉。每个进程都有一个进程组ID并被一个会话管理，通常一个会话开始于用户登录，终止于用户退出。setid有三个作用：
    • 让进程摆脱原会话控制
    • 让进程摆脱原进程组控制
    • 让进程摆脱原控制终端的控制
  3. 改变当前目录为根目录
  4. 重设文件权限
  5. 关闭原父进程打开的文件描述符

进程通信

管道

  1. 半双工共享内存
  2. 只能用于父子进程或兄弟进程（由于匿名性）

具名管道

使用路径名确定的管道，摆脱了匿名限制

  1. 可以使互不相关的两个进程通信
  2. 使用路径访问管道，通信的进程可以像读写文件一样访问管道
  3. 遵循FIFO原则，摆脱半双工限制

消息队列

由系统管理的消息链表

  1. 独立于收发进程存在
  2. 消除了同步的困难
  3. 接收端可以有选择的接受数据

共享内存

  1. 通信快速，而且效率高
  2. 没有提供同步机制

信号量

结合共享内存可以解决同步问题

进程调度

  1. FCFS先来先服务
  2. 短作业优先
  3. 优先级调度
  4. 高响应比优先
  5. 时间片轮转
  6. 多级队列调度（多优先级队列+时间片轮转）

进程的同步和互斥

  • 同步：进程之间需要按照一定的次序进行
  • 互斥：进程不能同时访问临界区

  3. 链式等待，循环等待
  4. 互斥条件，资源唯一
